A

Affacturage

Opération financière par laquelle une entreprise cède ses créances (factures) à un organisme spécialisé appelé « factor ». La RFE prévoit des types de documents dédiés (facture affacturée 393, avoir affacturé 396…) et des cadres de facturation spécifiques.

AIFE Agence pour l'Informatique Financière de l'État

Agence publique en charge du développement et de l'exploitation de Chorus Pro. Opère le PPF, l'annuaire central et le concentrateur de données pour le compte de la DGFIP.

AFNOR Association Française de Normalisation

Organisme officiel de normalisation. A publié les 3 normes techniques fondamentales : XP Z12-012 (formats), XP Z12-013 (API), XP Z12-014 (cas d'usage).

Annuaire

Répertoire centralisé administré par le PPF qui recense toutes les entreprises assujetties et leurs adresses de facturation électronique. Permet de savoir vers quelle PA envoyer une facture.

API Application Programming Interface

Interface de programmation entre systèmes. Deux API normalisées (Z12-013) : Directory Service (annuaire) et Flow Service (factures et statuts).

AS/4 Applicability Statement 4

Protocole de communication sécurisé pour l'échange de messages entre PA. Un des 3 protocoles de transport possibles (avec API REST et SFTP).

Assujetti (à la TVA)

Toute personne exerçant une activité économique indépendante. Critère fondamental : tous les assujettis en France sont soumis à la réforme.

Auto-facturation Self-billing

L'acheteur émet la facture pour le compte du vendeur. Codes spécifiques : 389 (facture), 261 (avoir), 500/501 (acompte). Nécessite un mandat du vendeur.

Avoir Note de crédit

Document annulant ou corrigeant une facture. Codes : 381 (standard), 261 (auto-facturé), 396 (affacturé), 503 (acompte). En UBL : schéma CreditNote.

B

B2B Business to Business

Transactions entre entreprises assujetties. Les factures B2B domestiques = cœur du périmètre e-invoicing : format structuré obligatoire.

B2C Business to Consumer

Transactions avec un particulier. Pas d'e-invoicing mais soumises au e-reporting (Flux 10).

B2G Business to Government

Transactions avec une entité publique. Transitent obligatoirement par Chorus Pro depuis 2020.

BG Business Group

Regroupement logique de données dans EN 16931. Ex : BG-4 = vendeur, BG-7 = acheteur, BG-25 = ligne de facture.

BR Business Rule

Règle de validation. BR-EN16931 = européennes, BR-FR = françaises. Ex : BR-FR-10 vérifie le SIREN dans l'annuaire.

BT Business Term

Donnée élémentaire du modèle EN 16931. BT-1 = n° facture, BT-2 = date, BT-23 = cadre de facturation, BT-112 = TTC.

C

CA3

Déclaration mensuelle/trimestrielle de TVA. La réforme vise le pré-remplissage automatique de la CA3 grâce aux données collectées.

Cadre de facturation

Donnée obligatoire (BT-23) qualifiant le contexte commercial : B1 (bien), S1 (service), M1 (mixte), B4/S4 (après acompte), S5 (sous-traitance), etc.

CDAR Cross Domain Acknowledgement and Response

Format UN/CEFACT pour les statuts de cycle de vie. Chaque changement de statut est transmis sous forme de message CDAR.

CGI Code Général des Impôts

Articles clés : 289 bis (e-invoicing), 290/290 A (e-reporting), 1788 E (sanctions).

Chorus Pro

Plateforme publique gratuite pour le B2G depuis 2017. Devient une PA à part entière. Héberge l'annuaire central et le concentrateur PPF.

CII Cross Industry Invoice

Format XML UN/CEFACT (version D22B). L'un des 3 formats acceptés. Base du volet structuré de Factur-X. Très utilisé en France/Allemagne.

CIUS Core Invoice Usage Specification

Adaptation nationale de EN 16931. La CIUS française ajoute des contraintes (cadre de facturation, SIREN, notes PMT/PMD/AAB…).

Clearance

Modèle dans lequel les factures sont vérifiées par une plateforme avant transmission. Le modèle français s'inspire du clearance (contrôles PA + données au PPF).

Code routage

Identifiant optionnel dans l'annuaire pour affiner le routage (par site, par service). Exemple : 12345678900012/COMPTA01.

CTC Continuous Transaction Controls

Contrôle continu des transactions en temps quasi réel. Le modèle français est un CTC : les données remontent à l'État en permanence.

Cycle de vie

Ensemble des statuts d'une facture (Déposée → Émise → Reçue → Approuvée/Refusée → Encaissée). Statuts transmis via messages CDAR.

D

DGFIP Direction Générale des Finances Publiques

Administration fiscale française, pilote de la réforme. Définit les spécifications, agrée les PA, exploite les données collectées.

E

EDI Échange de Données Informatisé

Mode d'échange structuré entre systèmes. L'un des 3 canaux possibles (avec API et portail).

E-invoicing

Échange obligatoire de factures structurées entre assujettis en France (B2B domestique + B2G). Couvre les Flux 1, 2 et 6.

E-reporting

Transmission des données des transactions hors e-invoicing : B2C, B2B international, paiements (TVA encaissement). Via le Flux 10.

EN 16931

Norme sémantique européenne : modèle de données (BT/BG), règles de validation (BR). Socle de tous les formats de la réforme.

ERP Enterprise Resource Planning

Progiciel de gestion intégré. Système source des factures, doit produire UBL/CII/Factur-X et se connecter à une PA.

ETI Entreprise de Taille Intermédiaire

Effectif < 5 000, CA < 1 500 M€. Obligation d'émission dès le 1er septembre 2026.

Exigibilité TVA

Moment où la TVA est due. Sur les débits (émission) ou à l'encaissement (paiement). Le statut « Encaissée » (212) déclenche l'exigibilité à l'encaissement.

F

Factur-X

Format hybride : PDF/A-3 lisible + XML CII embarqué. Profils : MINIMUM, BASIC WL, BASIC, EN16931, EXTENDED. Version actuelle : 1.07.3.

Facture d'acompte

Paiement partiel avant livraison complète. Code type 386 (ou 500 auto-facturée). Cadres B4/S4/M4 pour la facture définitive.

Facture rectificative

Corrige sans annuler. Code 384 (ou 471/472/473 pour les variantes). Doit référencer la facture initiale.

Flow Service

API normalisée (Z12-013) pour déposer et récupérer factures, statuts CDV et données e-reporting.

Flux (1 à 14)

14 types d'échanges entre acteurs. F1 = données réglementaires, F2 = facture, F6 = statuts, F7-9 = annuaire, F10 = e-reporting, F11 = retours PPF, F12 = DGFiP, F13-14 = annuaire technique. F4 et F5 n'existent pas (supprimés oct. 2024).

FNFE-MPE Forum National de la Facture Électronique

Association française de promotion de la facture électronique. Co-éditrice de Factur-X (avec le FeRD allemand).

G

GE Grande Entreprise

Effectif > 5 000 ou CA > 1 500 M€ + bilan > 2 000 M€. Émission obligatoire dès le 1er septembre 2026.

I

Immatriculation

Agrément DGFIP pour devenir PA. Après dossier conforme, tests d'interopérabilité et engagement d'audit. ~129 PA immatriculées ou en cours (mars 2026).

Interopérabilité

Capacité des systèmes (PA, PPF, ERP) à échanger des données de manière fluide grâce aux normes communes (Z12-012, Z12-013, Z12-014).

M

Mandat de facturation

Accord formel autorisant un tiers à émettre des factures au nom du vendeur. Prérequis pour l'auto-facturation et l'affacturage.

MDN Message Disposition Notification

Accusé de réception technique en AS/4. Distinct des statuts de cycle de vie métier.

Micro-entreprise

Effectif < 10, CA/bilan < 2 M€. Émission obligatoire au 1er sept. 2027 mais réception dès 1er sept. 2026.

N

Note de débit

Document de l'acheteur signalant un montant supplémentaire (pénalités, frais). Traitée via une facture spécifique avec cadre adapté.

O

OD / SC Opérateur de Dématérialisation / Solution Compatible

Prestataire non immatriculé. Ne peut pas transmettre directement au PPF. Agit comme intermédiaire technique auprès d'une PA.

P

PA Plateforme Agréée

Opérateur privé immatriculé par la DGFIP. Valide, transmet, gère les statuts et alimente l'annuaire. Chaque entreprise doit choisir au moins une PA.

PA-E PA d'Émission

Rôle émetteur : contrôles, Flux 1 au PPF (données), Flux 2 à la PA-R (facture complète).

PA-R PA de Réception

Rôle récepteur : reçoit le Flux 2, met à disposition du destinataire, transmet les statuts CDV.

PDP ⚠️ Ancien terme

Ancienne appellation des PA, utilisée dans les premières versions des textes. Le terme officiel est désormais Plateforme Agréée (PA).

PEPPOL Pan-European Public Procurement OnLine

Réseau européen d'échanges de documents commerciaux. Protocoles AS/4, annuaire SML/SMP. Canal complémentaire dans la réforme française.

PISTE

Portail d'API de l'État. Raccordement aux API de Chorus Pro et du PPF (annuaire, dépôt, statuts).

PME Petite et Moyenne Entreprise

Effectif < 250, CA < 50 M€. Émission obligatoire au 1er septembre 2027.

Portabilité

Droit de changer de PA à tout moment. L'ancienne PA assure la continuité et facilite la migration.

PPF Portail Public de Facturation

Plateforme publique : annuaire central, concentrateur de données, contrôles. Depuis oct. 2024, ne propose plus de service direct aux entreprises privées (sauf B2G via Chorus Pro).

R

Réception (obligation de)

Obligation pour toutes les entreprises dès le 1er sept. 2026. Implique d'avoir choisi une PA et d'être inscrit dans l'annuaire.

S

Schéma en Y

Architecture de la réforme : branche commerciale (Flux 2 : facture PA-E → PA-R) et branche fiscale (Flux 1 : données PA-E → PPF). Le « Y » = séparation des deux circuits.

SDI Sistema di Interscambio

Plateforme italienne de facturation électronique (modèle full clearance centralisé). Souvent citée comme référence. Le modèle français est décentralisé via les PA.

SFTP Secure File Transfer Protocol

Protocole sécurisé de transfert de fichiers. L'un des 3 modes de transmission (avec API REST et AS/4).

SIREN

Numéro unique à 9 chiffres (INSEE). Identifiant de base dans l'annuaire. Schéma 0002.

SIRET

SIREN + NIC (14 chiffres). Identifie chaque établissement. Schéma 0009. Permet un routage plus fin.

SML Service Metadata Locator

Composant Peppol (DNS) qui localise le SMP d'un participant.

SMP Service Metadata Publisher

Annuaire distribué Peppol : capacités de réception d'un participant. Distinct de l'annuaire centralisé PPF.

Statut obligatoire

Statut transmis au PPF : Déposée (200), Refusée (210), Encaissée (212), Rejetée (213). Les autres sont facultatifs.

T

TPE Très Petite Entreprise

= Micro-entreprise (effectif < 10). Émission sept. 2027, réception sept. 2026.

TVA Taxe sur la Valeur Ajoutée

Cœur de la réforme : moderniser la collecte TVA avec des données en temps quasi réel → pré-remplissage CA3.

U

UBL 2.1 Universal Business Language

Format XML OASIS. L'un des 3 formats acceptés. Schémas Invoice + CreditNote. Format natif Peppol, très utilisé en Europe.

V

ViDA VAT in the Digital Age

Initiative UE pour la TVA numérique. E-invoicing obligatoire intra-UE horizon 2030-2035. La réforme française s'y inscrit.

X

XSD XML Schema Definition

Fichier définissant la structure d'un XML. Des XSD de validation (profils BASE et FULL) sont fournis pour UBL et CII.

Z

ZUGFeRD

Format hybride allemand = Factur-X côté allemand. Même socle technique (PDF/A-3 + XML CII), mêmes profils.
