API Flow Service (Z12-013)

L'API principale pour déposer, rechercher et récupérer les flux de factures et de cycle de vie.

RouteMéthodeDescription
/flowsPOSTDéposer un flux (facture CII/UBL/Factur-X, CDV CDAR, e-reporting FRR)
/flows/searchPOSTRechercher des flux avec filtres multi-critères
/flows/{flowId}GETRécupérer un flux (Original, Converted, ReadableView, Metadata)
/healthcheckGETVérifier la disponibilité du service

Webhook (notification temps réel)

En complément de POST /search, un Webhook HTTP POST notifie en temps réel l'arrivée ou la modification d'un flux. Filtrable par FlowType, FlowDirection, FlowAckStatus.

3 protocoles de transport

📁

SFTP

Dépôt/retrait de fichiers. Auth bi-clé RSA + certificat X509v3. Chiffrement AES. Retrait sous 7j (purge auto).

Usage : flux batch volumineux, EDI historique

📡

AS/4

Push/Pull avec acquittement MDN. Signature SHA-2. Chiffrement AES 256. Encapsulation SOAP.

Usage : EDI pro, interopérabilité Peppol

🔗

API REST

HTTP(S) synchrone/asynchrone. JSON + XML. OAuth2 via PISTE. Codes retour standards.

Usage : intégration moderne, SaaS, temps réel

Cycle de vie d'une ligne d'annuaire

Création → En vigueur → En cours de clôture → Clôturée

Portabilité (changement de PA)

La nouvelle PA actualise les lignes, l'ancienne les clôture. L'ancienne PA doit fournir des services minimums pendant la transition (art. 1788 E du CGI).

Comment se connecter ?

1
S'inscrire sur PISTE (piste.gouv.fr)
2
Obtenir les clés d'accès (client_id, client_secret)
3
Récupérer un token Bearer via l'endpoint OAuth2
4
Appeler les APIs avec Authorization: Bearer {token}
